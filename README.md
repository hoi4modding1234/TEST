```

styles.css         ← 모든 스타일
```

9개 파일을 GitHub Pages 저장소의 **루트**에 업로드.

## 2. 새 기능 사용법

### 친구 시스템 — 편지로 신청하는 친구

이 앱의 정체성을 살린 디자인입니다. 친구가 되려면 진짜 편지를 주고받아야 해요.

**친구 신청 보내기:**
1. 편지 쓰기 → 받는 사람 입력 → **♥ 친구 신청 편지로 보내기** 토글을 켬
2. 평소처럼 편지 작성 (제목·본문·사진·우표 모두 자유)
3. 편지 보내기. 보내고 나면 봉투에 ♥ 친구 신청 배지가 붙음

**받는 쪽 흐름:**
1. 우편함을 열어 편지를 가져옴 (♥ 배지가 붙은 봉투)
2. 봉투를 뜯고 편지를 읽음
3. 편지 아래에 **"답신 쓰기"** 버튼이 나타남
4. 클릭하면 답신 편지 쓰기 화면으로 이동
5. 답신을 작성하고 보냄 (이때도 거리만큼 시간이 걸림)
6. 답신이 신청자에게 **도착하는 순간** 친구 관계가 확정됨

**친구 신청의 4단계:**
- 신청 발송 → 신청 도착 → 답신 발송 → 답신 도착 = 친구

가까운 도시면 몇 분이지만 멀리 사는 사람과는 친구가 되는 데 몇 시간이 걸려요. 그게 이 앱의 결입니다.

**거절·만료:**
- 거절 버튼은 없음. 답신 안 보내면 7일 후 자동 만료
- 신청 편지를 휴지통에 버리면 = 묵시적 거절
- 만료되면 7일간 재신청 불가

**친구 활용:**
- 편지 쓰기 화면 받는 사람 입력란 아래에 친구 칩이 떠 있어 한 번 클릭으로 입력
- 친구 탭에서 친구 목록 + 진행 중인 신청 한 번에 보기
- 친구 카드의 "편지 쓰기" 버튼으로 빠르게 편지 시작

### 편지 휴지통

받은 편지의 **🗑** 버튼 → 휴지통으로. 7일 후 영구 삭제. 그 전엔 휴지통 탭에서 복원 가능.

휴지통의 편지 옆에는 "○○ 후 삭제" 카운트다운이 표시됩니다.

## 3. 친구 신청 흐름 시각화

```
서울(나)                           뉴욕(상대)
  │                                    │
  │ ─── 친구 신청 편지 ───────────▶ │  (1시간 40분)
  │                                    │
  │                                    ★ 봉투 뜯음 (state: received)
  │                                    │
  │                                    │ "답신 쓰기" 클릭
  │                                    │
  │ ◀──── 수락 답신 편지 ─────────── │  (1시간 40분)
  │                                    │
  ★ 답신 도착 → 친구 관계 확정 ♥    │
```

빠른 도시(서울 ↔ 인천): 약 2~3분
중거리(서울 ↔ 부산): 약 36분
장거리(서울 ↔ 뉴욕): 약 3시간 20분

## 4. 자주 발생하는 문제

| 증상 | 원인 / 해결 |
|---|---|
| 친구 탭이 비어있음 | 아직 친구가 없음. 친구 신청을 보내거나 받아야 함 |
| ♥ 토글이 안 보임 | 받는 사람 입력 후에만 표시됨 (자기 자신·이미 친구·진행 중인 신청·쿨다운 시 숨김) |
| 답신 보냈는데 친구가 아님 | 답신이 거리만큼 도착해야 친구 확정. 보낸 편지함에서 도착 시각 확인 |
| 편지 휴지통 버튼이 안 보임 | 받은 편지에만 표시됨. 보낸 편지엔 없음 |
| 휴지통의 카운트다운이 안 줄어듦 | 새로고침 시 갱신됨. `purge_old_trash` 가 진입 시 자동 호출 |
| 친구 신청 편지를 휴지통에 넣으면? | 자동으로 신청 거절(`discarded`) 처리 + 7일간 재신청 불가 |

## 6. 처음 설치하시는 분 (v1~v6 다 안 거치셨다면)

위에서부터 차례로:

1. https://supabase.com 가입 → 새 프로젝트 (지역: Seoul)
2. SQL Editor에서 차례로 모두 RUN:
   - `setup.sql`
   - `migration_v2.sql`
   - `migration_v3.sql`
   - `migration_v4.sql`
   - `migration_v5.sql`
   - `migration_v6.sql`
   - `migration_v7.sql`
3. Authentication → Providers → Email → "Confirm email" 끄기 (테스트용, 권장)
4. Authentication → Email → "Leaked Password Protection" 토글 ON (보안)
5. Settings → API에서 Project URL 과 anon public 키 복사
6. `config.js` 의 두 상수에 붙여넣기
7. 9개 파일 모두 GitHub 저장소 루트에 업로드
8. Settings → Pages → main / (root) → Save
